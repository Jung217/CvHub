Abstract — 摘要

We propose CornerNet, a new approach to object detection where we detect an object bounding box as a pair of keypoints, the top-left corner and the bottom-right corner, using a single convolutional neural network. By detecting objects as paired keypoints, we eliminate the need for designing a set of anchor boxes commonly used in prior single-stage detectors. In addition to our novel formulation, we introduce corner pooling, a new type of pooling layer that helps the network better localize corners. Experiments show that CornerNet achieves 42.2% AP on MS COCO, outperforming all existing one-stage detectors.
我們提出 CornerNet，一種物件偵測的新方法，使用單一摺積神經網路將物件邊界框偵測為一對關鍵點——左上角與右下角。透過將物件偵測為成對的關鍵點，我們消除了先前單階段偵測器中常用的錨框設計需求。除了新穎的表述外，我們還引入了角點池化，一種幫助網路更好地定位角點的新型池化層。實驗顯示 CornerNet 在 MS COCO 上達到 42.2% AP，超越所有現有的單階段偵測器。
段落功能全文總覽——以「關鍵點偵測取代錨框」的核心創新統領全文。
邏輯角色摘要建立了「問題（錨框設計）-方案（角點配對）-工具（角點池化）-結果（42.2% AP）」的四層論述。
論證技巧 / 潛在漏洞「消除錨框」是極具吸引力的口號。但角點配對是否對所有物件形狀都有效（如極端長寬比），摘要未涉及。

1. Introduction — 緒論

The dominant paradigm in object detection relies on anchor boxes — a set of pre-defined boxes of various scales and aspect ratios that serve as detection candidates. Anchor-based detectors such as Faster R-CNN, SSD, and RetinaNet have achieved strong performance, but the design of anchor boxes introduces several challenges: the number, sizes, and aspect ratios of anchor boxes need to be hand-designed, and the large number of anchors creates a significant imbalance between positive and negative samples.
物件偵測的主流範式依賴於錨框——一組預定義的不同尺度與長寬比的框，作為偵測候選。基於錨框的偵測器如 Faster R-CNN、SSD 與 RetinaNet 已達到強勁效能，但錨框設計引入多項挑戰：錨框的數量、尺寸與長寬比需手動設計，且大量錨框造成正負樣本的嚴重不平衡。
段落功能背景鋪陳——指出錨框範式的系統性缺陷。
邏輯角色列舉三個知名偵測器建立錨框範式的普遍性，再精準指出其設計負擔與樣本不平衡問題。
論證技巧 / 潛在漏洞將「手動設計」與「樣本不平衡」並列為雙重問題，但 RetinaNet 的 Focal Loss 已在一定程度上緩解了不平衡問題。
We propose a fundamentally different approach: detecting objects as paired keypoints. Our CornerNet detects the top-left and bottom-right corners of each object and groups them using associative embedding. We introduce corner pooling, which looks along the boundary of the object to find the corner, since corners often lack local visual evidence.
我們提出一種根本不同的方法：將物件偵測為成對關鍵點。CornerNet 偵測每個物件的左上角與右下角，並使用關聯嵌入進行分組。我們引入角點池化，沿物件邊界尋找角點，因為角點通常缺乏局部視覺線索。
段落功能核心提案——CornerNet 的設計理念。
邏輯角色從問題自然過渡到解決方案：角點偵測 + 關聯嵌入 + 角點池化。
論證技巧 / 潛在漏洞「角點缺乏局部線索」的觀察精妙地引出角點池化。關聯嵌入借鑒自姿態估計領域。

2. Method — 方法

CornerNet uses a stacked hourglass network as the backbone. The network predicts two sets of heatmaps for top-left and bottom-right corners, along with embedding vectors for grouping and offsets for location refinement. Corners from the same object are grouped by the distance between their embedding vectors. We use a variant of focal loss for training, which reduces the penalty for negative locations near ground-truth corners.
CornerNet 使用堆疊沙漏網路作為骨幹。網路預測左上角與右下角的兩組熱力圖，以及用於分組的嵌入向量與用於位置精煉的偏移量。同一物件的角點透過嵌入向量距離分組。我們使用 focal loss 的變體進行訓練，降低對接近真實角點位置的負樣本懲罰。
段落功能網路架構——預測頭設計與訓練策略。
邏輯角色四個預測頭完整定義了從特徵到偵測結果的映射。
論證技巧 / 潛在漏洞修改版 focal loss 使角點定位有一定容忍度。沙漏網路的選擇與姿態估計一脈相承。
Corner pooling addresses the challenge that corners are often outside the object and lack local visual patterns. For a top-left corner, the corner pooling layer takes the maximum values from the right (horizontally) and from below (vertically), effectively looking along the object boundary. This operation provides strong localization signals even when the corner itself has no distinguishing features.
角點池化解決了角點通常位於物件外部且缺乏局部視覺模式的挑戰。對於左上角，角點池化層從右方（水平）與下方（垂直）取最大值，有效地沿物件邊界收集資訊。即使角點本身無顯著特徵，此操作也能提供強烈的定位信號。
段落功能角點池化的具體操作與動機。
邏輯角色角點池化是最具原創性的技術貢獻。
論證技巧 / 潛在漏洞從兩個方向取最大值的操作簡潔有效。但假設物件邊界有足夠視覺線索，對材質均勻物件可能效果有限。

3. Experiments — 實驗

On MS COCO test-dev, CornerNet achieves 42.2% AP, outperforming all one-stage detectors. Compared to RetinaNet (40.8% AP), CornerNet improves by 1.4% AP. Ablation studies show corner pooling contributes 2.0% AP, and the hourglass backbone outperforms ResNet-based alternatives.
在 MS COCO test-dev 上，CornerNet 達到 42.2% AP，超越所有單階段偵測器。相比 RetinaNet（40.8% AP），CornerNet 改善 1.4% AP。消融研究顯示角點池化貢獻 2.0% AP，沙漏骨幹優於 ResNet 替代方案。
段落功能主要實驗結果與消融分析。
邏輯角色以 COCO SOTA 驗證核心論點，消融量化了角點池化的獨立貢獻。
論證技巧 / 潛在漏洞2.0% AP 的角點池化貢獻證明設計必要性。但與兩階段偵測器的比較缺失。
Error analysis reveals that CornerNet produces many small incorrect bounding boxes, suggesting the network is better at detecting corners than grouping them. The inference speed is 244ms per image on a Titan X GPU, which is slower than some anchor-based detectors. However, the approach opens new possibilities for anchor-free detection.
誤差分析顯示 CornerNet 產生許多小的錯誤邊界框，表明網路偵測角點優於正確分組。推論速度為每張影像 244ms（Titan X GPU），慢於一些錨框偵測器。然而此方法為無錨框偵測開啟了新可能。
段落功能局限性分析——誠實指出弱點。
邏輯角色坦誠揭示角點分組挑戰與推論速度不足，增強可信度。
論證技巧 / 潛在漏洞主動誤差分析展現科學嚴謹性。「開啟新可能」將局限性轉化為研究方向。

4. Conclusion — 結論

We have presented CornerNet, a new approach that eliminates anchor boxes by detecting objects as paired keypoints. Our corner pooling enables effective corner detection even when corners lack local visual evidence. CornerNet achieves state-of-the-art one-stage detection on MS COCO. We believe this work demonstrates the viability of keypoint-based detection and will inspire future anchor-free methods.
我們提出了 CornerNet，透過將物件偵測為成對關鍵點消除錨框需求。角點池化即使角點缺乏局部視覺線索也能有效偵測。CornerNet 在 MS COCO 單階段偵測中達到最先進結果。我們相信此工作展示了基於關鍵點偵測的可行性，並將啟發未來的無錨框方法。
段落功能總結全文——重申無錨框偵測的可行性。
邏輯角色結論呼應摘要，強調 CornerNet 作為無錨框偵測範式開創者的歷史定位。
論證技巧 / 潛在漏洞「啟發未來」的預言在後續 CenterNet、FCOS 等工作中得到充分驗證。

作者核心主張（一句話）

CornerNet 透過將物件偵測重新表述為角點配對問題，消除了錨框設計需求，並以角點池化解決角點缺乏局部視覺線索的核心挑戰。

論證最強處

範式創新的開創性：首次成功將物件偵測從「錨框匹配」轉變為「關鍵點配對」，開啟無錨框偵測新時代。後續 CenterNet、FCOS、RepPoints 均受此啟發。角點池化是針對具體問題的優雅設計。

論證最弱處

推論效率與分組挑戰：244ms 的推論速度不夠實用。誤差分析揭示的角點分組問題是此範式的固有缺陷，CenterNet 後來透過偵測中心點來改善此問題。
